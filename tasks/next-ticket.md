# Feature: Video Workout Layout — End-to-End Activation

## Priority
High — Mobile UI already built, just needs backend + web integration to be usable.

## User Story
As a **trainer**, I want to assign the "Video" workout layout to my trainees so they see full-screen exercise demo videos while logging sets, creating an immersive coaching experience.

## Acceptance Criteria
- [ ] Backend: `WorkoutLayoutConfig.LayoutType` includes `'video'` as a valid choice
- [ ] Backend: Migration adds the new choice without data loss
- [ ] Backend: `GET /api/workouts/my-layout/` can return `layout_type: 'video'`
- [ ] Backend: `PATCH /api/trainer/trainees/{id}/layout-config/` accepts `layout_type: 'video'`
- [ ] Web: Layout config selector shows 4 options (Classic, Card, Minimal, Video)
- [ ] Web: Video option has appropriate icon, label, and description
- [ ] Web: Selecting Video persists to backend and shows as selected on reload
- [ ] Web: Exercise detail panel shows inline video player (not just YouTube link)
- [ ] Web: Video player loops, shows play/pause, has basic controls
- [ ] Mobile: When backend returns `layout_type: 'video'`, VideoWorkoutLayout renders

## Edge Cases
1. Exercise has no video_url — fallback gradient with muscle group color (already handled in mobile)
2. Exercise has YouTube URL — web player should handle both direct MP4 and YouTube embeds
3. Video fails to load — show error state, don't block workout logging
4. Trainer changes layout from video to classic mid-workout — layout only changes on next workout start
5. Old mobile app version doesn't know about 'video' layout — falls through to 'classic' default in switch statement

## Technical Approach

### Backend Changes
- **File:** `backend/trainer/models.py` — Add `VIDEO = 'video', 'Video'` to `LayoutType` TextChoices
- **Migration:** Auto-generated by `makemigrations`

### Web Changes
- **File:** `web/src/components/trainees/layout-config-selector.tsx` — Add 4th Video option
- **File:** `web/src/components/exercises/exercise-detail-panel.tsx` — Inline video player
- **New:** `web/src/components/exercises/exercise-video-player.tsx` — Reusable player (MP4 + YouTube)

### Mobile Changes
- None needed — already shipped

## Out of Scope
- Video preloading/caching optimization
- CDN/CloudFront setup
- Trainee self-service layout selection in mobile settings
